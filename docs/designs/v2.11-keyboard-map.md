# v2.11 Keyboard Shortcut Map

**Generated:** 2026-04-17
**Scope:** Every shortcut defined in `src/lib/keyboard/registry.ts` as of v2.11.0.
**Owner:** Each row is owned by the component bound via `useShortcut(id, …)`.

## Philosophy

Every action has a shortcut. `?` opens a discoverable overlay anywhere. Users can rebind anything in **Settings → Keyboard**. Defaults aim for muscle-memory compatibility with Superhuman / Linear / Granola where it makes sense, otherwise they prefer mnemonic over custom.

## Consistency rules (enforced by registry)

- `Cmd+Enter` = confirm / submit
- `Escape` = cancel / close
- `J / K` = next / previous in vertical lists
- `Cmd+K` = command palette
- `/` = search in a list context
- `Cmd+,` = Settings
- `?` = keyboard cheatsheet (always)
- Chord navigation: `G then <letter>` — tap G, then tap the destination within 1s

## Conflict-check gate

`scripts/check-shortcut-conflicts.ts` runs the registry against:

1. macOS system shortcuts (Cmd+Space, Cmd+Tab, screenshot) — **P0**
2. Electron menubar defaults (Cmd+Q, Cmd+W, Cmd+M) — **P1**
3. Standard editing (Cmd+C/V/X/Z/A/F) — **P2** (aware; we're not overriding)
4. Internal registry conflicts (two shortcuts, same binding, overlapping scope)

**Current status: clean** — 0 conflicts at any severity.

Run via:
```sh
bun scripts/check-shortcut-conflicts.ts
```

## IME / non-US keyboard fallback

- The matcher skips when `event.isComposing === true`, so any IME composition (Japanese, Korean, Chinese, Vietnamese) is immune.
- Chord shortcuts (`G then N`) are ignored in the notes-list scope when focus is on an input — the letter would otherwise collide with typing.
- All shortcuts can be rebound in **Settings → Keyboard**. Users on Dvorak, Colemak, or German ISO layouts can remap `J / K` to whatever feels natural to them without touching code.

## Known deferrals (v2.11.1 / v2.12)

- Shortcut for `commitment.snooze` currently binds `S` in `notes-list` scope but the handler isn't wired yet — snooze action exists but isn't exposed via shortcut. Wire in v2.11.1.
- Settings nav doesn't support `j/k` between sections. Keyboard-only path to the AI Models tab currently requires `Tab` navigation.
- `notes.next / prev` needs a focus ring when a note is keyboard-selected — today selection is implicit on click. v2.11.1 polish.
- Chord rebinding via the capture widget (Settings) isn't supported — chord shortcuts show a disabled kbd and can only be reset to default. Proper chord capture is a v2.12 enhancement.
